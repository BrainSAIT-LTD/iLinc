# Security Policy

## Supported Versions

We actively maintain and support the following versions of the `iLinc` project:

| Version   | Supported          |
|-----------|--------------------|
| `v1.x.x`  | ✅ Fully supported |
| `v0.x.x`  | ❌ No longer supported |

If you are using an unsupported version, we strongly recommend upgrading to the latest version to ensure you receive security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability in `iLinc`, please follow these steps:

1. **Do not publicly disclose the vulnerability.**
   - To protect our users, avoid sharing vulnerability details in public forums like GitHub Issues or Discussions.

2. **Report the vulnerability privately.**
   - Send an email to our security team at: **security@brainsait.com**.
   - Include the following details in your report:
     - A detailed description of the vulnerability.
     - Steps to reproduce the issue.
     - Potential impact or exploit scenarios.
     - Any recommended fixes (if available).

3. **Acknowledgment and Response.**
   - We will acknowledge receipt of your report within **48 hours**.
   - Our team will investigate the issue and provide a detailed response, including timelines for a fix or mitigation, within **5 business days**.

---

## Security Updates

We are committed to keeping `iLinc` secure. All security patches and updates will be communicated via:

- [GitHub Releases](https://github.com/BrainSAIT-LTD/iLinc/releases)
- A SECURITY.md update with resolved vulnerability details.
- Email notifications for subscribed users.

---

## Scope of Security Testing

We encourage responsible testing and disclosure within the following guidelines:

- **Allowed Activities:**
  - Testing for vulnerabilities in publicly available code and features.
  - Reporting security issues responsibly as outlined above.

- **Prohibited Activities:**
  - Exploiting vulnerabilities to access private user data.
  - Conducting tests that disrupt or degrade service.
